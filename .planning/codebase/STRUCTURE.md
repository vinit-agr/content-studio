## Directory Purposes

**`tools/remotion/src/`:**
- Purpose: Motion graphics rendering implementation
- Contains: React components, compositions, scenes, primitives, theme context
- Key files: `index.ts` (entry), `Root.tsx` (registration), `compositions/*/index.tsx` (top-level videos)

**`tools/remotion/src/primitives/`:**
- Purpose: Reusable visual building blocks
- Contains: Caption, Document, Chunk, Span, Token, MetricBar, Cursor, TitleCard
- Key files: `index.ts` (barrel export), each `.tsx` is a component
- Pattern: Each primitive is a React FC with typed props, uses `useCurrentFrame()` for animation, imports from shared theme

**`tools/remotion/src/compositions/`:**
- Purpose: Define video compositions (top-level Remotion registrations)
- Contains: Subdirectories per composition, each with index.tsx, frames.ts, scenes/, captions.ts, thumbnail.tsx
- Key files:
  - `[composition]/index.tsx`: Exports top-level component, uses Sequence to arrange scenes
  - `[composition]/frames.ts`: SCENES object defining timing, duration validation
  - `[composition]/captions.ts`: On-screen text per scene
  - `[composition]/scenes/NN-[name].tsx`: Individual scene components
  - `[composition]/thumbnail.tsx`: Still frame for thumbnails/YouTube

**`tools/remotion/src/theme/`:**
- Purpose: Local theme context provider
- Contains: ThemeContext (React Context), useTheme hook
- Key files: `ThemeContext.tsx`, `index.ts`
- Note: Actual theme values are in `shared/theme/`; this layer just provides access

**`shared/theme/`:**
- Purpose: Design system — single source of truth for colors, typography, animations
- Contains: Color palettes, font configuration, easing functions, type definitions
- Key files:
  - `colors.ts`: `defaultTheme` object with all color keys
  - `fonts.ts`: Font family names and `loadFonts()` function
  - `easings.ts`: Animation curves (fadeIn, slideIn, pulseDot, spanGlow)
  - `types.ts`: `Theme` and `ChunkIndex` TypeScript types
  - `projects/[project].ts`: Per-project theme overrides (spreads defaultTheme and overrides)
- Access: Imported as `@theme/*` path alias; components use `useTheme()` hook to consume

**`shared/assets/`:**
- Purpose: Static content and binaries
- Contains: Sample data, design assets (gitignored), documentation
- Key files: `data/sample-document.ts`
- Note: Binary assets have README.md; actual files are gitignored

**`projects/[source-repo]--[slug]/`:**
- Purpose: Per-project creative documentation
- Contains: Tool-agnostic script, storyboard, notes
- Key files: `script.md` (narrative), `storyboard.md` (scene breakdown), `notes.md` (iteration log)
- Pattern: One directory per video project; sync with canonical documentation in `docs/`

**`repo-references/`:**
- Purpose: Local clones of source repositories being explained in videos
- Gitignored; see README for expected clones
- Used by: Animators to reference real code while writing scripts

**`out/`:**
- Purpose: Rendered outputs
- Contains: MP4s (codec H.264), PNG thumbnails
- Pattern: `out/[project-slug]/[version].mp4` (e.g., `out/cx-agent-evals--chunk-vs-span/v01.mp4`)
- Gitignored; recreate via `pnpm render:[composition]`

**`log/`:**
- Purpose: Work log and iteration tracking
- Contains: Dated work logs, per-project iteration notes
- Pattern: New dated entry for each work session

**`docs/superpowers/`:**
- Purpose: Design specs and implementation plans
- Contains: Foundation design, phase plans, architecture docs
- Note: Referenced in README as source of truth for creative direction

## Naming Conventions

**Files:**

- **Components** (React functional components): PascalCase `.tsx`
  - Examples: `Caption.tsx`, `Document.tsx`, `TitleCard.tsx`

- **Utilities/Helpers** (non-component modules): camelCase `.ts`
  - Examples: `frames.ts`, `captions.ts`, `easings.ts`

- **Config files**: lowercase with dot-separated segments
  - Examples: `remotion.config.ts`, `tsconfig.json`, `.prettierrc`, `eslint.config.js`

- **Scene files**: `NN-[descriptive-name].tsx` (numeric prefix for order)
  - Examples: `01-intro.tsx`, `02-document.tsx`, `03-chunking.tsx`

**Directories:**

- **Feature directories**: kebab-case
  - Examples: `chunk-vs-span`, `cx-agent-evals--chunk-vs-span`

- **Technical directories**: lowercase plural
  - Examples: `primitives`, `compositions`, `scenes`, `theme`, `assets`

**Imports/Aliases:**

- `@shared/*`: Points to `shared/` directory
- `@theme/*`: Points to `shared/theme/` directory
- `@primitives`: Points to `tools/remotion/src/primitives/index.ts`
- `@primitives/*`: Points to individual primitives in `tools/remotion/src/primitives/`

**Variables/Functions:**

- **Variables**: camelCase (theme, frame, visibleText)
- **Functions**: camelCase (interpolate, loadFonts, useTheme)
- **Constants/Enums**: UPPER_SNAKE_CASE (SCENES, SAMPLE_DOCUMENT, CAPTIONS, FPS)
- **Type names**: PascalCase (Theme, ChunkIndex, DocumentProps, CaptionProps)

## Where to Add New Code

**New Composition (e.g., "my-video"):**
1. Create directory: `tools/remotion/src/compositions/my-video/`
2. Add files:
   - `index.tsx`: Main composition component exporting from here
   - `frames.ts`: SCENES object with timing, validation block
   - `captions.ts`: CAPTIONS object with on-screen text
   - `thumbnail.tsx`: Still frame component
   - `scenes/01-intro.tsx`, `02-...tsx`, etc.: Scene components
3. Import composition in `tools/remotion/src/Root.tsx`
4. Register in Root: `<Composition id="my-video" component={...} durationInFrames={...} fps={30} width={1920} height={1080} />`
5. Add npm script in `package.json`: `"render:my-video": "remotion render my-video out/.../v01.mp4"`

**New Primitive (e.g., "MyShape"):**
1. Create file: `tools/remotion/src/primitives/MyShape.tsx`
2. Export type `MyShapeProps` and component `MyShape`
3. Add to barrel export in `tools/remotion/src/primitives/index.ts`: `export { MyShape, type MyShapeProps } from './MyShape';`
4. Use in scenes via: `import { MyShape } from '@primitives';`

**New Shared Theme Asset:**
1. **For color palette overrides:** Create or edit `shared/theme/projects/[project-name].ts`
   - Pattern: Spread `defaultTheme` and override specific keys
   - Type-safe: Use `satisfies Theme` to ensure completeness
2. **For animation curves:** Add to `shared/theme/easings.ts`
3. **For fonts:** Add to `shared/theme/fonts.ts`, update `loadFonts()`

**New Data/Asset:**
1. Create file in `shared/assets/data/` (e.g., `my-data.ts`)
2. Export constant: `export const MY_DATA = { ... };`
3. Import in scenes: `import { MY_DATA } from '@shared/assets/data/my-data';`
4. If binary: Create README.md in same directory; add actual file to `.gitignore`

**New Scene (for existing composition):**
1. Create file: `tools/remotion/src/compositions/[composition]/scenes/NN-scene-name.tsx`
2. Use numeric prefix NN in order (01, 02, 03...)
3. Export named component: `export const SceneNameScene: FC = () => (...)`
4. Add to SCENES in frames.ts with start frame and duration
5. Add caption to captions.ts for that scene
6. Import and use in compositions/[composition]/index.tsx with `<Sequence>`

**New Project Documentation:**
1. Create directory: `projects/[source-repo-name]--[video-slug]/`
2. Add files:
   - `script.md`: Narrative/voiceover (canonical source)
   - `storyboard.md`: Scene breakdown with timings
   - `notes.md`: Iteration log and creative decisions
